## 一、写新文章（最常用的操作）

### 步骤

1. 打开文件夹：`C:\Users\yang\.qclaw\workspace-54nuktoh8cd83kjj\astro-tech-blog\src\content\blog\`
2. 在里面新建一个 `.md` 文件（文件名随意，英文即可，比如 `my-first-post.md`）
3. 用任何文本编辑器打开（记事本也行，推荐 VS Code），写入以下内容：

```markdown
---
title: "我的第一篇文章"
description: "这是我的第一篇技术文章"
pubDate: 2026-05-21
tags: [入门, 测试]
category: 随笔
---

## 大家好

这是我的第一篇文章，很高兴开始写博客！

### 我要分享的内容

- 第一点
- 第二点
- 第三点

正文随便写，跟微信排版差不多，就是用几个符号来表示格式。
```

4. 保存文件
5. 在终端运行：
```bash
cd C:\Users\yang\.qclaw\workspace-54nuktoh8cd83kjj\astro-tech-blog
npm run dev
```
6. 打开浏览器访问 http://localhost:4321 就能看到新文章了

### 就这么简单！核心就是：
- **文件顶部**那几行以 `---` 包围的是"信息区"，告诉博客标题、日期、标签等
- **下面的**就是正文，用 Markdown 语法写

## 二、Markdown 速查表（写文章唯一需要会的）

下面涵盖了 95% 你会用到的东西：

### 标题（用 # 号）
```
# 一级标题（文章内一般不用，因为标题已经在信息区写了）
## 二级标题（最常用）
### 三级标题
```

### 正文
```
直接写文字就是正文，段落之间空一行。
```

### 加粗 / 斜体
```
**加粗文字**
*斜体文字*
```

### 列表
```
- 无序列表项 1
- 无序列表项 2
- 无序列表项 3

1. 有序列表项 1
2. 有序列表项 2
3. 有序列表项 3
```

### 链接
```
[显示文字](https://网址)
```

### 图片
```
![图片描述](图片网址)
```

### 代码（行内）
```
用 `反引号` 包围就是代码
```

### 代码块
````markdown
```javascript
const a = 1;
console.log(a);
```
````

语言可以换成 `python`、`go`、`sql`、`bash` 等。

### 引用
```
> 这是一段引用文字
```

### 表格
```
| 列1 | 列2 | 列3 |
|-----|-----|-----|
| 数据 | 数据 | 数据 |
| 数据 | 数据 | 数据 |
```

### 分割线
```
---
```

## 三、信息区参数说明

每篇文章顶部 `---` 之间的内容：

```yaml
title: "文章标题"          # 必填，显示在卡片和页面顶部
description: "一句话描述"    # 必填，文章摘要
pubDate: 2026-05-21        # 必填，发布日期
tags: [标签1, 标签2]        # 选填，文章标签
category: 分类名            # 选填，默认"技术"
featured: false             # 选填，true 会在首页"精选"区显示
draft: false                # 选填，true 则不会发布（草稿）
heroImage: ""               # 选填，封面图片网址
```

**举例**：
```yaml
---
title: "Vue3 组合式 API 入门"
description: "从选项式到组合式，5分钟搞懂 Vue3 的核心变化"
pubDate: 2026-05-25
tags: [Vue, 前端, JavaScript]
category: 前端开发
featured: true
---
```

## 四、常见操作

### 预览博客
```bash
cd C:\Users\yang\.qclaw\workspace-54nuktoh8cd83kjj\astro-tech-blog
npm run dev
```
然后打开 http://localhost:4321

### 修改文章
直接编辑 `src/content/blog/` 下的 `.md` 文件，保存后浏览器会自动刷新。

### 删除文章
删除对应的 `.md` 文件即可。

### 暂存草稿
在信息区加 `draft: true`，文章就不会出现在博客上。
